# groupSectors_simulatedAnnealing_2 优化算法复杂度分析

## 一、优化策略概述

`groupSectors_simulatedAnnealing_2` 是对 `groupSectors_simulatedAnnealing_1` 的优化版本，主要优化策略包括：

1. **减少迭代次数**：降低温度循环次数和内层迭代次数
2. **增量能量计算**：只计算变化部分的能量，避免完整重算
3. **简化邻域生成**：优先使用 O(n) 的交换策略，减少 O(n²) 的随机移动
4. **提前终止机制**：连续多次不接受新解时提前退出
5. **优化初始解生成**：使用冲突列表加速，从 O(n³) 降到 O(n²)
6. **限制优化组迭代**：限制 optimizeGroups 的迭代次数

## 二、各阶段时间复杂度详细分析

### 1. 构建冲突图 (1639-1647 行)

```cpp
for (int i = 0; i < n; ++i) {
    for (int j = i + 1; j < n; ++j) {
        if (sectorsIntersect(sectors[i], sectors[j])) {
            adj[i][j] = adj[j][i] = true;
        }
    }
}
```

**时间复杂度**: O(n²)

- 与原版本相同，无变化

### 2. 构建冲突列表 (1770-1776 行)

```cpp
std::vector<std::vector<int>> conflict_lists(n);
for (int i = 0; i < n; ++i) {
    for (int j = 0; j < n; ++j) {
        if (adj[i][j]) {
            conflict_lists[i].push_back(j);
        }
    }
}
```

**时间复杂度**: O(n²)

- 一次性构建，后续查询为 O(1) 到 O(冲突数)

### 3. 优化的初始解生成 (1778-1810 行)

#### 3.1 排序阶段

```cpp
std::sort(sector_order.begin(), sector_order.end(), ...)
```

**时间复杂度**: O(n log n)

#### 3.2 优化的贪心分配

```cpp
for (int idx : sector_order) {  // n 次循环
    for (int g = 0; g < num_groups_initial; ++g) {  // 最多 n 组
        for (int j : conflict_lists[idx]) {  // 平均冲突数 k << n
            if (current_solution[j] == g) {
                conflicts++;
            }
        }
    }
}
```

**时间复杂度**: O(n² × k)

- 外层循环: n 次
- 中层循环: 最坏情况 n 组，但实际通常远小于 n
- 内层循环: 只检查冲突列表，平均冲突数 k 通常远小于 n
- **实际复杂度**: O(n²) (当 k 为常数时) 到 O(n³) (最坏情况)
- **相比原版本**: 从 O(n³) 优化到接近 O(n²)

### 4. 优化的模拟退火主循环 (1825-1940 行)


#### 4.2 内层循环次数

#### 4.3 每次迭代的复杂度

**策略 2: 随机移动 (20%概率)**

- **相比原版本**: 从 O(n²) 优化到 O(n × k)，**实际减少 70-90%**
